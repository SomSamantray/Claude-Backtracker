# claude-session-tracker — GitHub Packaging Design Spec

**Date:** 2026-04-01  
**Author:** Som  
**Status:** Reviewed & revised (post spec-review)

---

## Overview

Package the `/backtrack` Claude Code slash command as a public GitHub repository (`claude-session-tracker`) so any Claude Code user can install it in one command. On install, the tool pre-populates a cache of the user's last 50 sessions, creates a human-readable session archive, and sets up activity logging. Works on Windows, Mac, and Linux.

---

## Repository Structure

```
claude-session-tracker/
├── README.md                          ← install guide, usage, requirements
├── LICENSE                            ← MIT license
├── package.json                       ← name, version, node engine requirement
├── install.js                         ← cross-platform Node.js installer/updater
└── commands/
    └── backtrack.md                   ← portable /backtrack command template
```

**No runtime dependencies.** Uses only Node.js built-ins (`fs`, `os`, `path`, `readline`).

## Files Generated on User's Machine

All files live under `~/.claude/backtrack/`:

| File | Created by | Purpose |
|---|---|---|
| `~/.claude/commands/backtrack.md` | Installer | The active slash command (tokens replaced per user) |
| `~/.claude/backtrack/cache.json` | Installer | Session metadata + AI summaries (lazy) |
| `~/.claude/backtrack/sessions-archive.log` | Installer | Human-readable snapshot of 50 sessions at install time |
| `~/.claude/backtrack/activity.log` | /backtrack command | Append-only usage log |
| `~/.claude/backtrack/info.json` | Installer (written LAST) | Version, install date, last update — absence = incomplete install |

**Atomicity strategy:** `info.json` is always written last. If a previous install was interrupted, `~/.claude/backtrack/` exists but `info.json` is absent → treated as a first-install and all files are re-created.

## `cache.json` Schema (explicit)

```json
{
  "_offset": 0,
  "_version": "1.0.0",
  "sessions": {
    "<sessionId>": {
      "summary": null,
      "display": "First message text...",
      "project": "C:\\Users\\Som\\Documents\\VibePro-CLI",
      "firstTimestamp": 1773657113310,
      "lastTimestamp": 1773665629844
    }
  }
}
```

- `_offset`: pagination state (updated at runtime by `/backtrack`)
- `_version`: cache format version for future migrations
- `sessions`: object keyed by `sessionId`
- `summary`: `null` until generated by `/backtrack`; after generation, a string of ≤12 words
- `firstTimestamp`: lowest timestamp seen for this sessionId (used for summary input)
- `lastTimestamp`: highest timestamp seen for this sessionId (used for sort order; updated by update run)

## `install.js` — Installer Logic

### Execution Order

**Step 0 (first, always): Node.js version check**
```
const [major] = process.versions.node.split('.').map(Number);
if (major < 16) { error and exit }
```

**Step 1: Path setup using `path.join()` (never string concatenation)**
```js
const home = os.homedir();
const claudeHome = path.join(home, '.claude');
const backtrackDir = path.join(claudeHome, 'backtrack');
const commandsDir = path.join(claudeHome, 'commands');
```
All subsequent paths are built with `path.join()`.

**Step 2: Verify Claude Code is installed**
```js
if (!fs.existsSync(claudeHome)) { error: "Claude Code not found" }
```

**Step 3: Determine mode (first install vs update)**
```js
const infoPath = path.join(backtrackDir, 'info.json');
const isUpdate = fs.existsSync(infoPath);
const isInterruptedInstall = fs.existsSync(backtrackDir) && !isUpdate;
// Both isUpdate=false cases (first install + interrupted) go through first-install flow
```

### First Install Flow

1. Create `~/.claude/backtrack/` with `fs.mkdirSync(backtrackDir, { recursive: true })`
2. **Create `~/.claude/commands/` if it doesn't exist:** `fs.mkdirSync(commandsDir, { recursive: true })`
3. Read `commands/backtrack.md` from repo, replace all three tokens (see Tokens section), write to `commandsDir/backtrack.md`
4. Parse `history.jsonl` → build session list (see History Parsing section)
5. Take last 50 sessions
6. Write `cache.json` with 50 session stubs (all `summary: null`)
7. Write `sessions-archive.log` (see Log Formats section)
8. Write empty `activity.log`
9. **Check for old `backtrack-sessions.json` → migrate if present** (see Migration section)
10. **Write `info.json` LAST** — its presence signals a complete installation

### Update Flow (re-running `node install.js` when `info.json` exists)

1. Create `~/.claude/commands/` if it doesn't exist (may have been deleted)
2. Re-read `commands/backtrack.md` from repo, **replace tokens again**, write to commands dir
3. Read existing `cache.json` → collect all known sessionIds
4. Parse `history.jsonl` → find sessions NOT in cache (new sessions)
5. For sessions already in cache: **update `lastTimestamp`** if their latest history entry has a higher timestamp (handles resumed sessions moving to top of recency list)
6. Append new sessions to `cache.json` (all `summary: null`)
7. Append new session entries to `sessions-archive.log` (with `[pending: <sessionId>]` placeholders — see below)
8. Update `info.json.lastUpdatedAt`

### Error Handling

| Condition | Message |
|---|---|
| Node.js < 16 | `Node.js 16+ required. Current: vX.x. Please upgrade: https://nodejs.org` |
| `~/.claude/` not found | `Claude Code not found at <path>. Install Claude Code first: https://claude.ai/code` |
| `history.jsonl` missing | `No session history found. Use Claude Code for a while, then re-run this installer.` |
| `history.jsonl` empty | Same as above |
| Permission denied on write | `Cannot write to <path>. Check folder permissions and try again.` |
| Interrupted prior install | `Incomplete installation detected. Cleaning up and reinstalling...` then re-runs first-install |

## History Parsing (`history.jsonl`)

Each line is a JSON object: `{ display, timestamp, project, sessionId }`.

Parse all lines. For each unique `sessionId` build:
```
sessions[sessionId] = {
  display: entry with LOWEST timestamp's display field  (summary input)
  firstTimestamp: lowest timestamp seen
  lastTimestamp: highest timestamp seen  (sort key)
  project: entry with lowest timestamp's project field
}
```

Sort by `lastTimestamp` descending. Take first 50.

**Note on display quality (expected ~67% poor on average):** A display value is considered poor quality and will trigger fallback summarization if it:
- Starts with `/` (slash command)
- Is under 15 characters
- Starts with `[Pasted`

The fallback (reading the session JSONL for the first real user message) is the expected common path, not the exception.

## Template Tokens in `backtrack.md`

| Token | Replaced with | Example (Windows) | Example (Mac) |
|---|---|---|---|
| `{{CLAUDE_HOME}}` | `path.join(home, '.claude')` | `C:\Users\john\.claude` | `/Users/john/.claude` |
| `{{USER_HOME}}` | `os.homedir()` | `C:\Users\john` | `/Users/john` |
| `{{BACKTRACK_DIR}}` | `path.join(home, '.claude', 'backtrack')` | `C:\Users\john\.claude\backtrack` | `/Users/john/.claude/backtrack` |

Token replacement must be done for the update run as well (not just first install).

## Path Encoding (for session JSONL lookup)

Claude Code encodes project paths into directory names by replacing every non-alphanumeric character with a single dash. Examples:
- `C:\Users\Som` → `C--Users-Som` (`:` and `\` each → `-`)
- `C:\Users\Som\Documents\Tech_Integrations - Mio (1)` → `C--Users-Som-Documents-Tech-Integrations---Mio--1-`

**Robust resolution strategy (used by both slash command and installer):**
1. List all directories in `~/.claude/projects/`
2. For the given `project` value, encode it: replace `[^a-zA-Z0-9]` with `-`
3. If exact match found → use it
4. Else: find the directory whose name is most similar to the last path component (folder name) of the project path
5. Construct session file path: `<matched-dir>/<sessionId>.jsonl`
6. If file still not found → graceful fallback (no crash)

## Log File Formats

### `sessions-archive.log`

```
=== Claude Session Archive ===
Generated: Apr 01, 2026 at 10:45 AM
Sessions: 50 | Summaries generated on first /backtrack run

────────────────────────────────────────────────────────────
[1]  Apr 01, 2026  10:45 AM  |  Documents\Claude-Sessions_Tracker
     ID: e5d48377-6c38-4b8c-acd5-ed876937a951
     Summary: [pending: e5d48377-6c38-4b8c-acd5-ed876937a951]
     First message: Go to Documents/Claude-Sessions_Tracker folder...

[2]  Mar 31, 2026  07:02 AM  |  Documents\Easter-Egg
     ID: 02535541-f461-49a1-ae9f-b390ccd46c08
     Summary: [pending: 02535541-f461-49a1-ae9f-b390ccd46c08]
     First message: Go to Documents/Easter-Egg folder...
────────────────────────────────────────────────────────────
```

**Critical design note:** Each `[pending: <sessionId>]` placeholder is globally unique in the file because it contains the sessionId. This enables the `/backtrack` command to use the Edit tool's `old_string` parameter to target exactly one line per replacement — avoiding the Edit tool's uniqueness requirement failure.

**Back-fill process (done by `/backtrack` command):**
- After generating summaries (Step 4 of backtrack.md), for each newly summarized session:
  - Use Edit tool with `old_string: "[pending: <sessionId>]"` → `new_string: "<summary text>"`
  - This targets exactly one line (guaranteed unique due to sessionId)
- All 10 back-fills happen sequentially as 10 Edit tool calls — acceptable given this is a one-time operation per session

### `activity.log`

Append-only. Written by `/backtrack` command using a Write-append pattern (read file, append line, write back).

```
[Apr 01, 2026 11:00 AM] LISTED sessions 1-10 (offset: 0)
[Apr 01, 2026 11:01 AM] LOADED session #3 — Documents\VibePro-CLI (id: d66da164)
[Apr 01, 2026 11:30 AM] LISTED sessions 1-10 (offset: 0)
[Apr 02, 2026 09:15 AM] LISTED sessions 11-20 (offset: 10) [more]
[Apr 02, 2026 09:16 AM] LOADED session #14 — Documents\Testing (id: b13032f2)
```

### `info.json`

```json
{
  "version": "1.0.0",
  "installedAt": "2026-04-01T10:45:00.000Z",
  "lastUpdatedAt": "2026-04-01T10:45:00.000Z",
  "platform": "win32",
  "claudeHome": "C:\\Users\\Som\\.claude"
}
```

## `/backtrack status` Output Format

When `$ARGUMENTS` is `"status"`, Claude reads the three files and displays:

```
╔══════════════════════════════════════════╗
║   claude-session-tracker  v1.0.0        ║
╚══════════════════════════════════════════╝

Installed:       Apr 01, 2026
Last updated:    Apr 01, 2026
Platform:        win32

Cache:           50 sessions total
                 → 47 with AI summaries
                 → 3 pending (run /backtrack to generate)
Activity log:    12 LISTED calls · 8 LOADED calls
Archive:         50 sessions

Run /backtrack to browse sessions.
```

Reading logic:
- `info.json` → version, installedAt, lastUpdatedAt, platform
- `cache.json` → count keys in `sessions` object; count entries where `summary !== null`
- `activity.log` → count lines containing "LISTED" and count lines containing "LOADED"

## Migration from Old Cache Format

Som's existing `~/.claude/backtrack-sessions.json` uses this schema:
```json
{
  "_offset": 0,
  "d66da164-...": {
    "summary": "...",
    "display": "...",
    "project": "...",
    "firstTimestamp": 123,
    "lastTimestamp": 456
  }
}
```

Migration rules (run only if `cache.json` does NOT already exist):
1. Read `backtrack-sessions.json`
2. Skip the `_offset` key
3. For each sessionId key: map the object directly into `cache.json.sessions[sessionId]`
4. Set `_offset` from old file into new `cache.json._offset`
5. Set `_version: "1.0.0"` in new file
6. Write `cache.json`
7. Do NOT delete old `backtrack-sessions.json` (harmless, user can delete manually)

## Changes to `backtrack.md` vs Current Installed Version

| Change | Reason |
|---|---|
| All `C:\Users\Som\.claude` → `{{CLAUDE_HOME}}` | Cross-platform portability |
| `C:\Users\Som\` stripping → `{{USER_HOME}}\` stripping | Per-user short path |
| Cache path: `backtrack-sessions.json` → `{{BACKTRACK_DIR}}\cache.json` | New location |
| Cache schema: flat root → `sessions` sub-object | Cleaner schema, enables status queries |
| **New**: append to `activity.log` on list view | Activity tracking |
| **New**: append to `activity.log` on session select | Activity tracking |
| **New**: back-fill `sessions-archive.log` when summaries generated (Edit per sessionId) | Archive stays current |
| **New**: handle `$ARGUMENTS == "status"` | Version/stats display |
| **New**: back-compat: on startup, check for old `backtrack-sessions.json` and note migration path | Smooth transition |

## `README.md` Structure

```markdown
# claude-session-tracker

Browse and resume your previous Claude Code sessions with the /backtrack command.

## Requirements
- Node.js 16+
- Claude Code installed and used (needs session history)

## Install
```bash
git clone https://github.com/<username>/claude-session-tracker
cd claude-session-tracker
node install.js
```

## Usage
In any Claude Code session:
- `/backtrack` — list your last 10 sessions with AI summaries
- Type `1`–`10` to load that session's context
- `/backtrack more` — go further back (10 at a time)
- `/backtrack status` — show version and usage stats

## Update
```bash
cd claude-session-tracker
git pull
node install.js
```
Re-running install.js detects the existing installation and only adds new sessions.

## Uninstall
```bash
rm ~/.claude/commands/backtrack.md
rm -rf ~/.claude/backtrack/
```
Windows: use `del` and `rmdir /s` equivalents.
```
